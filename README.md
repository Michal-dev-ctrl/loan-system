# מערכת השכרת ציוד לאירועים – גמ"ח אור לכלה שמחת "יום טוב"

מערכת ווב לניהול השכרת ציוד, פיקדון, תרומה, חיפוש והחזרת ציוד עם מעקב נזקים.

---

## איך לשמור / לייצא את הפרויקט ללקוח

### אפשרות 1: העתקת התיקייה (הכי פשוט)
1. **העתיקי את כל תיקיית הפרויקט** (למשל `loan-system`) למחשב אחר או לארכיון.
2. **בלי התיקייה `node_modules`** – לא חובה להעביר אותה (משקלת הרבה). בלקוח או אצלך מחדש מריצים `npm install` (ראי למטה).
3. **מה כן להעביר:** כל הקבצים והתיקיות מלבד `node_modules`:
   - `src/`
   - `public/`
   - `package.json`
   - `package-lock.json` (אם קיים)
   - `tsconfig.json`
   - `postcss.config.mjs` / קבצי קונפיגורציה אחרים שבשורש הפרויקט

### אפשרות 2: קובץ ZIP
1. צרי קובץ ZIP של **כל** תיקיית הפרויקט **חוץ מ־`node_modules`** (לחצי ימני על התיקייה → "דחיסה" / "Compress", או השתמשי ב־7-Zip וכללי "exclude" ל־node_modules).
2. שמרי את הקובץ (למשל `loan-system-export.zip`) ושמרי גם גיבוי במקום נוסף.
3. ללקוח: לחלץ את ה־ZIP, לפתוח טרמינל בתיקייה ולהריץ `npm install` ואז `npm run build` ו־`npm start` (או לפרוס לשרת – ראי "פריסה לאינטרנט").

### אפשרות 3: Git (לשימוש מקצועי ועדכונים עתידיים)
1. התקיני Git אם עדיין אין.
2. בתיקיית הפרויקט:
   - `git init`
   - הוסיפי קובץ `.gitignore` עם שורה: `node_modules`
   - `git add .` ואז `git commit -m "גרסה ראשונה"`
3. צרי מאגר (repository) ב־GitHub או GitLab והעלי את הקוד (`git remote add origin ...` ואז `git push`).
4. כך יש לך גיבוי בענן, ואפשר להמשיך לעדכן את הפרויקט ולמסור ללקוח גישה (או קובץ ZIP מגרסה מסוימת).

## הרצה במחשב (פיתוח ובדיקה)

```bash
# התקנת חבילות (פעם ראשונה, או אחרי שקיבלת את הפרויקט בלי node_modules)
npm install

# הרצה לבדיקה מקומית – נפתח דפדפן ב־http://localhost:3000
npm run dev
```

## בניית גרסה לפריסה (production)

```bash
npm run build
npm start
```

אחרי `npm start` האתר רץ מקומית בפורט 3000. אפשר לפרוס את התיקייה (כולל `.next` שנוצר ב־build) לשרת שמריץ Node.js.

## איך להוסיף שינויים בעתיד

1. **פתחי את הפרויקט ב־Cursor (או VS Code):**  
   File → Open Folder → בחרי את תיקיית `loan-system`.

2. **ערכי קבצים:**  
   כל הקוד נמצא תחת `src/`:
   - עמוד ראשי: `src/app/page.tsx`
   - עמודי השכרה: `src/app/rental/` (פרטים, פיקדון, תאריכים, ציוד, סיכום)
   - חיפוש: `src/app/search/page.tsx`
   - החזרה: `src/app/return/[id]/page.tsx`
   - עיצוב גלובלי: `src/app/globals.css`
   - Header עם לוגו: `src/app/components/AppHeader.tsx`
   - קטלוג פריטים: `src/app/rental/items/page.tsx` ו־`catalog.ts`

3. **אחרי שינויים:**  
   - לבדיקה: `npm run dev`  
   - לגרסה מעודכנת ללקוח: `npm run build` ואז להעלות/לפרוס מחדש (או לשלוח ZIP מעודכן).

4. **גיבוי:**  
   לפני שינויים גדולים – העתיקי את כל התיקייה (בלי `node_modules`) או עשי commit ב־Git, כדי שאפשר תמיד לחזור לגרסה שעובדת.
